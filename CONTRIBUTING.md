# Contribuir

Una contribución debe preservar el enfoque conceptual. No se acepta una nueva implementación si solo traduce tokens sin explicar sus diferencias semánticas o idiomáticas.

## Lista de comprobación

- El concepto está definido sin depender de un lenguaje.
- Existe pseudocódigo o contrato equivalente.
- Los casos de prueba son comunes.
- La implementación es idiomática.
- Las diferencias están clasificadas.
- Los comandos son reproducibles.
- No se afirma una equivalencia que el lenguaje no ofrece.
